## Implementation Details

### Data Fetching
- Courses: useAsyncData('courses-catalog', () => queryCollection('courses').order('order', 'ASC').all())
- Lesson counts: useAsyncData('courses-lesson-counts') — single lessons query grouped in-memory
- Progress: useFetch<CourseProgress[]>('/api/progress/courses', { immediate: false })
  - Only fetched when loggedIn === true (client-side watch trigger)

### Auth Pattern
- const { loggedIn } = useUserSession()  // from @onmax/nuxt-better-auth
- Progress fetch gated behind loggedIn check
- Client-side watch with immediate:true handles session hydration

### Nuxt UI v4 Components Used
- UCard: course card container with hover shadow/lift
- UBadge: difficulty indicator (color: success|warning|error)
- UButton: "Start Course" / "Continue" CTA (color=primary, variant=soft)
- UIcon: clock, book-open, arrow-right, academic-cap

### Design System Compliance
- Colors: text-secondary, text-tertiary-default (matches existing article/Card.vue patterns)
- Spacing: gap-6 grid, p-4 card padding, consistent with container mx-auto pattern
- Typography: font-bold text-3xl heading, text-sm descriptions, text-xs meta
- Hover: grayscale→color image transition (matches article/Card.vue)
- Responsive: grid-cols-1 md:grid-cols-2 lg:grid-cols-3 (per spec)

### Route Coexistence
- courses/index.vue → /courses (catalog)
- courses/[...slug].vue → /courses/:slug (course/lesson detail — unchanged)
- Nuxt prioritizes index.vue for exact /courses match

### Content Schema Fields Used (from content.config.ts courses collection)
- title, description (required strings)
- difficulty: 'beginner' | 'intermediate' | 'advanced'
- duration: number (minutes)
- tags: string[] (shown, max 4)
- order: number (sort order)
- image: string optional (cover)

### API Contract (from server/api/progress/courses.get.ts)
Response: Array<{
  courseSlug: string
  status: string
  completedLessons: number
  totalLessons: number
  lastAccessedAt: number | null  // epoch ms
}>
