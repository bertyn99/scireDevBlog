## Design Decisions

1. **SkillMastery data source**: Derived from review-queue response (concepts with mastery values). Keeps to the 4 required endpoints.
2. **Streak computation**: Client-side from activity timestamps, allows today-not-yet-active streak continuity.
3. **Layout uses `bg-default`/`text-muted`/`border-default`**: Nuxt UI v4 semantic tokens (not hardcoded colors) — adapts to dark mode.
4. **Auth**: `definePageMeta({ middleware: 'auth' })` — existing `auth.ts` middleware checks `useUserSession().loggedIn` and redirects to `/auth/login?redirect=...`.
5. **No new dependencies added**.
